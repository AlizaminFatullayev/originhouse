Project Memory Bank — Origin House Coffee

Project Overview

This project is a premium coffee shop and coffee e-commerce front-end website based on an existing 5-page HTML/CSS design generated by Stitch AI.

The goal is not to redesign the website from scratch. The goal is to preserve the current visual design as much as possible and build correct, clean, scalable front-end functionality on top of it.

Project name: Origin House Coffee
Project type: Premium coffee shop website + coffee e-commerce prototype
Main goal: Turn the static design into a functional, responsive, portfolio-ready front-end project.

Main Design Direction

The website should feel:

* Premium
* Minimal
* Warm
* Editorial
* Modern
* Clean
* Spacious
* Coffee-inspired
* Similar in feeling to high-end coffee brands like WatchHouse

The existing Stitch AI design should be treated as the visual source of truth.

Do not randomly change the design, layout, colors, typography, spacing, or visual identity unless it is necessary to fix responsiveness, accessibility, or broken layout.

* Header navigation should work correctly.

Preferred Tech Stack

Use the current project structure if it already exists.

If conversion is needed, use:

* React
* Vite
* React Router
* CSS Modules or clean CSS
* Context API or Zustand for cart state
* localStorage for cart persistence

Do not add backend.

Do not add real payment.

Do not add unnecessary libraries unless they are useful.

Framer Motion can be used only for subtle animations if it does not break the existing design.

Code Quality Rules

* Preserve the design.
* Do not rewrite everything from scratch unless absolutely necessary.
* Use reusable components.
* Avoid duplicate code.
* Keep file structure clean.
* Use meaningful component names.
* Use meaningful CSS class names.
* Make the UI accessible.
* Use semantic HTML.
* Add alt text to images.
* Buttons should be real buttons.
* Links should be real links.
* Avoid inline styles unless necessary.
* Avoid unnecessary complexity.

Milestone Workflow

The project should be completed in milestones.

After every milestone:

1. Run the project.
2. Check for errors.
3. Fix errors before moving forward.
4. Commit changes to GitHub with a clear commit message.
5. Continue to the next milestone only after the current milestone is stable.

Each commit should represent one meaningful milestone.

Use clear commit messages such as:

* milestone: initialize project structure
* milestone: implement routing and layout
* milestone: build shop and product filtering
* milestone: implement product detail page
* milestone: add cart sidebar functionality
* milestone: build locations page and modal
* milestone: complete responsive mobile menu
* milestone: polish accessibility and final UI fixes
